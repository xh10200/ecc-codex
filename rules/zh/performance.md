# 性能优化

## 模型选择策略

**GPT-5.4-mini**：
- 高频轻量代理
- 探索、分类和摘要
- 多代理系统中的低成本 worker 角色

**GPT-5.4**：
- 主要开发工作
- 编排多代理工作流
- 复杂编码、调试和审查任务

## 上下文窗口管理

避免在上下文窗口的最后 20% 进行以下操作：
- 大规模重构
- 跨多个文件的功能实现
- 调试复杂交互

上下文敏感度较低的任务：
- 单文件编辑
- 独立工具创建
- 文档更新
- 简单 bug 修复

## 推理与规划

对于需要深度推理的复杂任务：
1. 在可用时选择更强模型或更高 reasoning effort
2. 在大范围改动前先分阶段拆解任务
3. 对高风险改动做多轮复审
4. 使用分角色子代理获得多样化视角

## 构建排查

如果构建失败：
1. 使用 **build-error-resolver** 代理
2. 分析错误消息
3. 增量修复
4. 每次修复后验证
